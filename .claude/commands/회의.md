# /회의 — AI 직원 회의 소집

안건: $ARGUMENTS

너는 **비서실장**으로서 회의를 사회 본다. 아래 절차를 따른다.

## 절차

1. **참석자 선정**: 사용자가 참석자를 지정했으면 그대로, 아니면 안건과 가장 관련 깊은 직원 2~3명을 선정한다 (마케팅PD·상담실장·운영매니저·콘텐츠작가·의학팩트체커·경영비서).
2. **사전 자료 읽기**: `CLAUDE.md`와 참석 직원들의 에이전트 파일(`.claude/agents/`), 안건과 관련된 위키 문서를 읽는다.
3. **1라운드 — 관점 발표**: 참석 직원을 각각 서브에이전트로 호출해, 자기 전문 영역 관점에서 원인 진단과 구체적 방안을 발표하게 한다. **서로 다른 관점·우선순위가 드러나야 한다.**
4. **2라운드 — 반박·보완**: 각 직원에게 다른 직원의 발언을 전달하고, 동의하는 부분과 이견을 밝히며 반박·보완하게 한다.
5. **결론 — 비서실장 정리**: 합의된 방향을 실행 순서대로 정리하고, 남은 이견은 숨기지 말고 "대표 판단 필요"로 명시한다. 직원별 액션 아이템을 배정한다.

## 회의록 저장

- 위치: `02_AI_Wiki/meetings/YYYY-MM-DD_안건요약.md`
- frontmatter: `status: draft`, `updated`, `owner: AI사서`, `tags: [회의록, 관련팀]`
- 구성: 안건 / 참석 / 라운드별 발언 요약 / 합의 사항 / 남은 이견 / 액션 아이템(담당·기한)
- `meetings_MOC.md` 에 등록하고, 액션 아이템이 특정 위키 문서와 연결되면 `[[링크]]`를 건다.

## 규칙

- 발언은 실제 회의 말투로, 구체적으로 (추상적인 "노력하자" 금지).
- 의학적 주장이 나오면 출처 원칙(CLAUDE.md §3)을 따른다.
- 마지막에 사용자에게 핵심 합의·이견·액션 아이템을 요약해 보고한다.
