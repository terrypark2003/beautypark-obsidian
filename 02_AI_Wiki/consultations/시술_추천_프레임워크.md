# 시술 추천 프레임워크

고민 → 시술 제안까지의 표준 순서. **핵심: 시술을 먼저 말하지 않는다. 진단이 먼저다.**

## 4단계

1. **고민 확정**: 고객 언어로 — "가장 신경 쓰이는 것 하나만 꼽으면?" → [[conditions_MOC]]의 어느 노트인지 매핑
2. **원인 설명**: 왜 이런 상태인지 (근거는 [[procedures_MOC]] verified 내용만)
3. **선택지 2~3개 제시**: 각각 (기대 경과 / 횟수 / 다운타임 / 비용대) — 1개만 제시하면 강매로, 4개 이상은 혼란으로 느낀다
4. **우선순위 권고**: "저라면 이 순서로 하겠다" + 이유. 최종 선택은 고객에게

## 금지

- 근거 없는 매칭 ("요즘 이게 유행이라") · 예산 초과 패키지 우선 제시 · 효과 보장
